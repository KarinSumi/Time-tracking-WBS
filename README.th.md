# Aion Enterprise Time Logger

🌐 **เลือกภาษา**: [English](README.md) | ภาษาไทย

ยินดีต้อนรับสู่ **Aion Enterprise Time Logger** — แพลตฟอร์มการจัดการพอร์ตโฟลิโอและระบบบันทึกเวลาปฏิบัติงานขององค์กร ที่ออกแบบมาเพื่อการบันทึกชั่วโมงงานอย่างแม่นยำ การวางแผนโครงการเชิงกลยุทธ์ การวิเคราะห์กำลังคน และการบริหารจัดการระดับองค์กร

## ⚡ การติดตั้งอย่างรวดเร็วใน 1 บรรทัด (Windows, Linux, Mac)

กรุณาตรวจสอบว่ามี **Node.js** และ **Git** ติดตั้งอยู่ในเครื่องของคุณ จากนั้นเปิดเทอร์มินัลแล้วคัดลอกคำสั่งด้านล่างนี้ไปรันเพื่อติดตั้งและเริ่มต้นใช้งานระบบโดยอัตโนมัติ:

### Linux / macOS:
```bash
git clone https://github.com/KarinSumi/Time-tracking-WBS.git && cd Time-tracking-WBS && node setup.js
```

### Windows (PowerShell):
```powershell
git clone https://github.com/KarinSumi/Time-tracking-WBS.git; cd Time-tracking-WBS; node setup.js
```

*สคริปต์อัตโนมัตินี้จะทำการติดตั้งไลบรารีทั้งหมด คัดลอกฐานข้อมูลจำลอง (Sample DB) และคอมไพล์โปรเจกต์ให้พร้อมรันทันที*

## 💾 ทดลองใช้งานทันทีด้วยฐานข้อมูลตัวอย่าง (Sample Database)

หากต้องการรีเซ็ตฐานข้อมูลกลับไปยังสถานะตั้งต้นสำหรับการทดสอบ (ที่มีข้อมูลโครงการ โครงสร้าง WBS การบันทึกชั่วโมงงานจำลอง และสมาชิกทีมครบถ้วน):

1. **คัดลอกฐานข้อมูลตัวอย่าง**:
   ```bash
   npm run db:restore-sample
   ```
2. **รันเซิร์ฟเวอร์สำหรับพัฒนา**:
   ```bash
   npm run dev
   ```

## 🎥 บทเรียนการใช้งานและขั้นตอนปฏิบัติทีละขั้นตอน

ด้านล่างนี้คือ 8 โมดูลการทำงานหลัก ซึ่งประกอบด้วยไฟล์ภาพเคลื่อนไหวสาธิตพร้อมรายละเอียดขั้นตอนการคลิกเพื่อรันตามวิดีโอ:

### 1. การสมัครสมาชิกและสร้างบัญชี (ผู้ใช้ทุกคน)
ลงทะเบียนองค์กร สร้างบัญชีผู้ใช้ใหม่ และเข้าสู่ระบบ

![Account Registration](frontend/public/tutorial/assets/01_register_account.webp)
🎬 **[ชมวิดีโอสาธิตการใช้งาน (รูปแบบ MP4)](frontend/public/tutorial/assets/01_register_account.mp4)**

#### ขั้นตอนปฏิบัติ:
1. คลิกที่ลิงก์ **Register** บริเวณด้านล่างของหน้าเข้าสู่ระบบ
2. กรอกข้อมูล **ชื่อเต็ม (Full Name)**, **อีเมล (Email)**, **ชื่อองค์กร (Organization Name)** และ **รหัสผ่าน (Password)**
3. อ่านรายละเอียดและกดยอมรับเงื่อนไขในช่อง **Terms of Service & Privacy Policy**
4. คลิกปุ่มสีน้ำเงิน **Create Account** เพื่อสร้างบัญชี
5. ระบบจะนำคุณกลับมายังหน้าเข้าสู่ระบบ กรอกอีเมลและรหัสผ่านที่สมัครใช้งาน จากนั้นคลิก **Sign In**

---

### 2. การใช้งานแดชบอร์ดหลัก (ผู้ใช้ทุกคน)
ทำความเข้าใจภาพรวมการแสดงผล ปฏิทินลงเวลา และงานที่ได้รับมอบหมาย

![Dashboard Navigation](frontend/public/tutorial/assets/02_login_dashboard.webp)
🎬 **[ชมวิดีโอสาธิตการใช้งาน (รูปแบบ MP4)](frontend/public/tutorial/assets/02_login_dashboard.mp4)**

#### ขั้นตอนปฏิบัติ:
1. ตรวจสอบหน้าจอแดชบอร์ดหลักที่แสดง **เวลาสะสมรวมประจำสัปดาห์ (Timesheet widgets)**
2. ตรวจสอบส่วน **งานที่ได้รับมอบหมาย (Active Tasks)** เพื่อดูภารกิจปัจจุบันของคุณ
3. ใช้แผงควบคุม **บันทึกเวลาด่วน (Quick Log)** บนแดชบอร์ดเพื่อลงเวลางานประจำวันอย่างรวดเร็ว
4. ดูตาราง **ปฏิทินรายสัปดาห์ (Weekly Visual Calendar)** เพื่อดูความหนาแน่นของการทำงานในแต่ละวัน
5. ใช้เมนูด้านซ้ายในการนำทางไปยังหัวข้อต่าง ๆ เช่น โครงการ (Projects), แผนงาน (Plans), ทีมงาน (Team), รายงาน (Reports) และการตั้งค่าระบบ (Admin)

---

### 3. การสร้างโครงการและเฟสงาน (ผู้ดูแลระบบ)
สร้างแผนงานหลักและแบ่งระยะเวลาการทำงาน (Phases)

![Project Setup](frontend/public/tutorial/assets/03_project_setup.webp)
🎬 **[ชมวิดีโอสาธิตการใช้งาน (รูปแบบ MP4)](frontend/public/tutorial/assets/03_project_setup.mp4)**

#### ขั้นตอนปฏิบัติ:
1. ไปที่หน้า **Projects** ผ่านทางเมนูด้านซ้าย
2. คลิกปุ่ม **Add Project** สีน้ำเงินบริเวณมุมบนขวา
3. กรอก **ชื่อโครงการ (Project Name)** (เช่น `Stitch Dashboard`) และเลือกสีประจำโครงการ จากนั้นคลิก **Save**
4. คลิกแท็บ **Phases** หรือปุ่มจัดการระยะเวลาของโครงการ
5. คลิกปุ่ม **Add Phase** กรอกชื่อระยะการทำงาน (เช่น `Build`) จากนั้นคลิก **Save**

---

### 4. โครงสร้างการแบ่งงาน - WBS (ผู้ดูแลระบบ)
สร้างแผนงานย่อยแบบลำดับขั้นและการแสดงผลบนแผนภูมิแกนต์ (Gantt Chart)

![Task Planning](frontend/public/tutorial/assets/04_task_planning.webp)
🎬 **[ชมวิดีโอสาธิตการใช้งาน (รูปแบบ MP4)](frontend/public/tutorial/assets/04_task_planning.mp4)**

#### ขั้นตอนปฏิบัติ:
1. ไปที่หน้า **Plans** ผ่านทางเมนูด้านซ้าย
2. เลือก **โครงการ (Project)** และ **ระยะการทำงาน (Phase)** ที่ต้องการวิเคราะห์จากตัวเลือก
3. คลิกปุ่ม **Add Task** เพื่อสร้างหัวข้องานใหม่ในโครงสร้างแบ่งงาน (WBS)
4. ระบุ **รายละเอียดงาน (Task Description)**, **วันเริ่มต้น/สิ้นสุด (Start/End Date)**, **ชั่วโมงที่วางแผน (Planned Hours)** และเลือกผู้รับผิดชอบ
5. หากต้องการจัดกลุ่มแบบลำดับชั้น ให้ระบุ **งานหลัก (Parent Task)** ในตัวเลือก (เช่น นำงาน `UI Components` ไปเป็นงานย่อยภายใต้ `Project Foundation`)
6. คลิกปุ่ม **Save** ผังโครงการจะแสดงผลแบบลำดับขั้นควบคู่กับ **แผนภูมิแกนต์ (Gantt Chart)** ทันที

---

### 5. การนำเข้ารายชื่อสมาชิกทีม (ผู้ดูแลระบบ)
ลงทะเบียนพนักงานและสมาชิกทีมจำนวนมากผ่านไฟล์ Excel

![Team Onboarding](frontend/public/tutorial/assets/05_bulk_upload.webp)
🎬 **[ชมวิดีโอสาธิตการใช้งาน (รูปแบบ MP4)](frontend/public/tutorial/assets/05_bulk_upload.mp4)**

#### ขั้นตอนปฏิบัติ:
1. ไปที่หน้า **Team** ผ่านทางเมนูด้านซ้าย
2. คลิกปุ่ม **Bulk Register** หรือไปยังส่วนอัปโหลดเพื่อเข้าสู่ฟังก์ชันนำเข้าไฟล์ข้อมูล
3. คลิกปุ่ม **Download Template** เพื่อรับไฟล์เทมเพลต Excel สำหรับกรอกข้อมูล
4. เปิดไฟล์แล้วกรอกข้อมูลพนักงาน ได้แก่ **ชื่อ (Name)**, **อีเมล (Email)**, **บทบาท (Role)** และ **อีเมลหัวหน้า (Manager Email)**
5. บันทึกไฟล์เทมเพลต จากนั้นกลับมาหน้าเว็บเพื่อเลือกไฟล์และคลิกปุ่ม **Upload**
6. ตรวจสอบตารางรายชื่อทีมงานที่จะอัปเดตและแสดงผลความสัมพันธ์การขึ้นตรงต่อหัวหน้างานอย่างถูกต้อง

---

### 6. การบันทึกชั่วโมงงานและส่งคำขออนุมัติ (ผู้ใช้ทุกคน)
การระบุรายละเอียดงานประจำวันและส่งแบบร่างชั่วโมงทำงานประจำสัปดาห์

![Time Logging](frontend/public/tutorial/assets/06_time_logging.webp)
🎬 **[ชมวิดีโอสาธิตการใช้งาน (รูปแบบ MP4)](frontend/public/tutorial/assets/06_time_logging.mp4)**

#### ขั้นตอนปฏิบัติ:
1. ในหน้าแดชบอร์ดหลัก ค้นหาหน้าต่างบันทึกเวลาด่วน **Quick Log Time**
2. เลือก **โครงการ (Project)**, **ระยะการทำงาน (Phase)** และ **ชิ้นงาน (WBS Planned Task)** ที่ระบุในแผนงาน
3. กรอกจำนวน **ชั่วโมงปฏิบัติงาน (Worked Hours)** (เช่น `4` ชั่วโมง) และระบุบันทึกรายละเอียดงานสั้น ๆ
4. คลิกปุ่ม **Log Time** บันทึกดังกล่าวจะขึ้นแสดงในระบบเป็นสถานะแบบร่าง (**DRAFT**)
5. ไปที่หน้า **Time Logs** เลือกรายการแบบร่างที่ต้องการส่งทางด้านหน้า แล้วกดปุ่ม **Submit** เพื่อขออนุมัติอย่างเป็นทางการ

---

### 7. การวิเคราะห์ภาพรวมและตารางความหนาแน่นงาน (ผู้ดูแลระบบ)
การประเมินชั่วโมงปฏิบัติงานตามแผนเทียบกับงานจริง และแผนภูมิความร้อนกำลังคน

![Analytics Reports](frontend/public/tutorial/assets/07_analytics_reports.webp)
🎬 **[ชมวิดีโอสาธิตการใช้งาน (รูปแบบ MP4)](frontend/public/tutorial/assets/07_analytics_reports.mp4)**

#### ขั้นตอนปฏิบัติ:
1. ไปที่หน้า **Reports** และคลิกแท็บ **Capacity**
2. เลือกช่วง **วันเริ่มต้น (Start Date)** และ **วันสิ้นสุด (End Date)** เพื่อกำหนดสัปดาห์หรือช่วงเวลาที่ต้องการดู
3. สังเกตแถบตารางวิเคราะห์ความร้อน **Resource Heatmap**:
   - 🟩 **ช่องสีเขียว**: ปริมาณงานปกติ มีการใช้งานกำลังคนในเกณฑ์ปกติ
   - 🟥 **ช่องสีแดง**: มีภาระงานหนักเกินไป (>8 ชั่วโมงต่อวันทำงาน)
   - ⬛ **ช่องสีเทา**: วันหยุดสุดสัปดาห์หรือวันหยุดของบริษัท
4. คลิกปุ่ม **Export Report** มุมบนขวาเพื่อส่งออกข้อมูลตารางกำลังคนในรูปแบบ CSV

### 8. การตั้งค่าระบบและการเปลี่ยนแบรนด์ขององค์กร (ผู้ดูแลระบบระดับสูง)
อัปโหลดโลโก้ ปรับเปลี่ยนสีธีมหลักตามเอกลักษณ์องค์กร

![Superadmin Customization](frontend/public/tutorial/assets/08_superadmin_config.webp)

#### ขั้นตอนปฏิบัติ:
1. ไปที่หน้า **Admin Settings** (หรือ Admin) ผ่านทางเมนูด้านซ้าย
2. ค้นหาหัวข้อควบคุมการตกแต่งแบรนด์ **Branding Customization**
3. กรอกรหัสสีหลักขององค์กรที่คุณต้องการปรับ (เช่น รหัสสีส้ม `#ff5722` หรือเลือกจากแถบสี)
4. ใช้เมนูอัปโหลดไฟล์ในการเปลี่ยนภาพ **โลโก้ของบริษัท (Logo image)**
5. คลิกปุ่ม **Save Customization** สีสันของปุ่มควบคุมและโลโก้หลักในเมนูด้านซ้ายจะเปลี่ยนตามอัตลักษณ์องค์กรทันที
